# 《GeoEvolve》
## 一、 项目概述
1. **游戏类型**：2D像素风动作冒险（类银河恶魔城）。
2. **核心玩法**：几何形态进化（三角形 $\rightarrow$ 正方形 $\rightarrow$ 多边形 $\rightarrow$ 圆形），利用形态特有的物理属性与“缺口”匹配机制进行战斗与地图解谜。

## 二、 技术栈
1. **游戏引擎**：Unity 6000.0.63f1。
2. **美术资产**：Aseprite。
3. **版本控制**：Git & GitHub。

## 三、 美术与视觉基调规范
建立明确的视觉反差，不仅能增强画面表现力，也能极大提升玩家对“形状匹配”机制的直观理解。
1. **背景环境**：废弃的工厂管道、巨大的齿轮、粗糙的金属表面。环境应尽量保持暗调和低饱和度。
2. **角色与交互物**：主角的不同形态、敌人的核心“缺口”、可破坏的机关。利用高饱和度的色彩在暗调工业背景中脱颖而出，提供极强的视觉引导。

## 四、 开发里程碑规划

###  **M1：核心原型期** 目标：验证“好不好玩”
在这个阶段，不需要任何精美的美术资产，全部使用 Unity 自带的 2D 基础图形（白模）。
**程序**：

1. 搭建基础的 2D 物理角色控制器（移动、跳跃、土狼时间、跳跃缓冲）。
2. 实现形态切换状态机（State Machine），完成“三角形（高机动/穿透）”与“正方形（重力/阻挡）”的无缝切换。
3. 开发基础的“缺口匹配”碰撞检测逻辑（Tag 或 Layer 判定）。

**策划**：

1. 在纸面或白板上画出核心机制的交互逻辑图。
2. 设计 2-3 种基础几何怪物的行为树（AI）。
   
### **M2：垂直切片期** 目标：确定最终品质标杆
完成一个包含完整游戏循环的小关卡，这将作为后续量产的模板。

**美术**：

1. 在 Aseprite 中绘制主角（三角、方形）的待机、移动、攻击、受击动画。
2. 绘制一套工业风的 Tilemap（地面、墙壁、背景层次）。

**程序**：

1. 使用 Shader Graph 制作形态切换的过渡特效。
2. 加入屏幕震动（Camera Shake）、顿帧（Hit Stop）和手柄震动，死磕“硬核”的动作打击感。

**策划**：

1. 使用 Tilemap 搭建一个测试房间：包含出生点、几个杂兵、一个需要“形态解谜”的机关，以及一个小型 Boss 战。

### **M3：内容量产期** 目标：扩充世界观
基于 M2 的模板，开始像搭积木一样铺设内容。

1. 系统扩展：加入五边形、六边形、圆形的能力逻辑（如射击、护盾、持续滚动）。
2. **关卡设计**：设计经典的 Metroidvania 地图结构：主区域 $\rightarrow$ 遇到无法通过的障碍（如圆形管道） $\rightarrow$ 探索分支 $\rightarrow$ 击败 Boss 获得新形状 $\rightarrow$ 回来解锁新区域。
3. 数据管理：完善 ScriptableObjects 架构，集中管理怪物血量、玩家伤害、技能 CD 等数值，方便随时调整。

### **M4：打磨与发布准备**

1. 系统：完成主菜单、设置界面、存档系统、UI 界面（血条、形态能量槽）。
2. 音频：导入并配置 BGM 与音效（UI 音效、物理碰撞声、进化音效）。
3. 优化：性能分析（Profiler），优化 Tilemap 渲染批次，确保 Shader 和粒子效果在目标平台（如 WebGL 或 PC）上流畅运行。
